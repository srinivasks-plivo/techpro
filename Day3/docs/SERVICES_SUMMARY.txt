═══════════════════════════════════════════════════════════════════════════════
                       SERVICE #1: REDIS SERVICE
═══════════════════════════════════════════════════════════════════════════════

WHAT: Store temporary session data while call is active

WHY: Need FAST access to current call state
     - Can't query database every millisecond (slow)
     - Can't hardcode in Python (not flexible)
     - Redis = in-memory, microseconds

HOW: JSON stored in Redis with 30-minute TTL

KEY METHODS:
├─ create_session()      → Start new call session
├─ get_session()         → Retrieve current state
├─ update_session()      → Save changes
├─ add_user_input()      → Record digit press
├─ set_current_menu()    → Navigate to new menu
├─ mark_call_completed() → Mark as successful
├─ mark_call_abandoned() → Mark as abandoned
└─ delete_session()      → Cleanup

SESSION DATA EXAMPLE:
{
    "call_uuid": "4f3a4e5c-b2d8-4a7f",
    "from_number": "+15551234567",
    "current_menu_id": "main_menu",
    "menu_history": ["main_menu", "sales_menu"],
    "user_inputs": [
        {"menu_id": "main_menu", "digit": "1", "timestamp": "..."},
        {"menu_id": "sales_menu", "digit": "1", "timestamp": "..."}
    ],
    "start_time": "2026-01-29T10:00:00Z",
    "state": "active"
}

═══════════════════════════════════════════════════════════════════════════════
                       SERVICE #2: PLIVO SERVICE
═══════════════════════════════════════════════════════════════════════════════

WHAT: Generate XML responses that control voice calls

WHY: Plivo doesn't execute Python. It only understands XML.
     We respond with XML, Plivo executes it.

HOW: Build XML strings with different elements for different actions

KEY METHODS:
├─ generate_menu_xml()          → Menu with digit input
├─ generate_transfer_xml()      → Transfer to phone number
├─ generate_hangup_xml()        → End call
├─ generate_speak_only_xml()    → Just say something
├─ generate_play_xml()          → Play audio file
├─ generate_invalid_input_xml() → Invalid digit error
├─ generate_timeout_xml()       → No input error
└─ validate_xml()               → Check XML is valid

PLIVO XML ELEMENTS:
├─ <Response>      - Root element (required)
├─ <Speak>         - Text-to-speech message
├─ <GetDigits>     - Wait for digit input
├─ <Dial>          - Transfer call
├─ <Hangup>        - End call
├─ <Play>          - Play audio file
├─ <Redirect>      - Send to different URL
└─ <Wait>          - Silence

XML EXAMPLE 1 - Menu with Input:
<Response>
  <GetDigits action="/webhooks/input" timeout="5" numDigits="1">
    <Speak>Press 1 for Sales, 2 for Support</Speak>
  </GetDigits>
</Response>

XML EXAMPLE 2 - Transfer to Operator:
<Response>
  <Dial timeout="30">
    <Number>+15551234567</Number>
  </Dial>
</Response>

XML EXAMPLE 3 - Goodbye and Hangup:
<Response>
  <Speak>Thank you for calling. Goodbye.</Speak>
  <Hangup />
</Response>

═══════════════════════════════════════════════════════════════════════════════
                        SERVICE #3: IVR SERVICE
═══════════════════════════════════════════════════════════════════════════════

WHAT: Orchestrate the entire IVR call flow

WHY: Ties together Redis, Plivo, and Models in a logical sequence

HOW: Three main methods handle the three call events

KEY METHODS:
├─ handle_incoming_call()  → Call arrives: create session, load menu
├─ handle_digit_input()    → Digit pressed: validate, update, navigate
└─ handle_hangup()         → Call ends: save to database, cleanup

METHOD 1: handle_incoming_call(call_uuid, from_number, to_number)
   Flow:
   1. Create Redis session
   2. Load main_menu from database
   3. Generate XML (via Plivo Service)
   4. Return XML to Plivo
   
   Result: Plivo speaks main menu to caller

METHOD 2: handle_digit_input(call_uuid, digit)
   Flow:
   1. Get session from Redis
   2. Get current menu from database
   3. Validate digit
   4. Record input in Redis
   5. Determine next menu
   6. Generate XML response
   7. Update Redis session
   8. Return XML to Plivo
   
   Result: Plivo either plays next menu or executes action

METHOD 3: handle_hangup(call_uuid, hangup_cause, duration)
   Flow:
   1. Get session from Redis
   2. Mark call as completed in session
   3. Save session to CallLog table (PERMANENT)
   4. Update CallerHistory table (PERMANENT)
   5. Delete session from Redis (CLEANUP)
   
   Result: Data now permanently stored in database

═══════════════════════════════════════════════════════════════════════════════
                     HOW SERVICES WORK TOGETHER
═══════════════════════════════════════════════════════════════════════════════

CALL TIMELINE:
1. 📞 Incoming Call
   ↓
   IVR Service.handle_incoming_call()
   ├─ Redis Service.create_session()
   ├─ Models: Load MenuConfiguration
   └─ Plivo Service.generate_menu_xml()
   ↓
   Return XML → Plivo speaks menu
   ↓
   
2. 🔘 User Presses Digit
   ↓
   IVR Service.handle_digit_input()
   ├─ Redis Service.get_session()
   ├─ Models: Load MenuConfiguration
   ├─ Validate digit
   ├─ Redis Service.add_user_input()
   ├─ Redis Service.set_current_menu()
   └─ Plivo Service.generate_menu_xml() [or transfer/hangup]
   ↓
   Return XML → Plivo executes action
   ↓
   (User presses another digit... loop back to step 2)
   ↓
   
3. 📵 Call Ends
   ↓
   IVR Service.handle_hangup()
   ├─ Redis Service.get_session()
   ├─ Models.save_to_CallLog()
   ├─ Models.update_CallerHistory()
   └─ Redis Service.delete_session()
   ↓
   Data now PERMANENT in PostgreSQL

═══════════════════════════════════════════════════════════════════════════════
                         SPEED VS. DURABILITY
═══════════════════════════════════════════════════════════════════════════════

TEMPORARY (Redis) - FAST ⚡⚡⚡
├─ Session data
├─ Microsecond response times
├─ Auto-expires (TTL)
└─ Lost if Redis crashes (OK for temporary)

PERMANENT (PostgreSQL via Models) - DURABLE 💾💾💾
├─ CallLog (detailed call records)
├─ CallerHistory (caller profiles)
├─ MenuConfiguration (menu structure)
├─ Millisecond response times
└─ Survives crashes (permanent)

THE HANDOFF:
Call is active:
├─ Redis stores everything (fast)
│
Call ends:
├─ Copy session → CallLog (permanent)
├─ Update CallerHistory (permanent)
├─ Delete Redis session (cleanup)
│
Result: Fast during call, permanent after call
